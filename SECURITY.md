# Security Policy

## Reporting a vulnerability

Please report security vulnerabilities **privately** to **hi@zevs.gg**. Do not open a public issue
for security problems.

Include a description, reproduction steps, the affected version/commit, and the impact. You'll get an
acknowledgement as soon as possible and a fix or mitigation timeline once the report is triaged.

## Supported versions

zynk has no published binary release yet; security fixes land on `main`. Build from source (or via Nix)
to get the latest fixes — see the [README](./README.md).
